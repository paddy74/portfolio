When the contract award transitioned in August of 2020 to a new prime &mdash; one that did not subcontract [IDSI](./idsi.md) &mdash; I followed the transition to accept a job with one of the new sub-contractors: Booz Allen Hamilton. This included a promotion for me from a junior to mid-level developer &mdash; but I was the _only_ member of the team who accepted the transition.

While this left me in an awkward position being that I was still early in my career &mdash; it also afforded me a unique opportunity to acquire a significant amount of experience in a short amount of time. Never being one to shy away from taking the initiative, I spent my first few months setting the foundation to onboard a new team and retain development momentum: rigourously documenting everything I knew about the project and its client, establishing project management guidelines, recreating all DevOps implementations (CI/CD, issue tracking, etc.), redeploying cloud infrastructure, and meeting my own pre-existing development milestones.

Coupled with numerous delays in finding an experienced project manager and hiring additional team members I soon advanced from a standard mid-level full-stack developer to to someone comfortably tackling all stages of the project management and software development lifecycle, onboarding and mentoring new team members, and regular reporting to contract and client leadership. Here I discovered what it was to be a software engineer and a leader, putting me on what is now my career track as a technical lead.

## Job Description

As a **Senior Software Engineer** and **Deputy Project Manager** supporting Army T2COM G2, I developed full-stack data management, discovery, analytics, and visualization systems and threat-actor network analysis tools, accelerating the creation of training and doctrine research publications.

- Designed and built T2COM G2's Training & Doctrine Research Data Platform in Azure GovCloud (IL4) using a microservice-based architecture; implemented ETL/data pipelines, data versioning, natural-language search, and full-stack web-applications; built with C#, ASP.NET Core MVC, ReactJS, Python, FastAPI, and Azure SQL Server.

- Developed OEVis ([https://oe.t2com.army.mil/oevis/](https://oe.t2com.army.mil/oe-vis-operational-environment-visualization/)), a full-stack graph-network analytics and simulation tool with real-time co-authoring, cloud-sync, and custom "bring-your-own-model" analytics model execution; replacing outdated commercial tools and cutting licensing costs; built with Python, FastAPI, ReactJS, PostgreSQL, and Neo4j.

- Enabled data governance including auditing, lineage, and version control through metadata stores backed by custom Cosmos DB and Azure Blob Storage extensions.

- Built data ingestion and transformation workflows using Python, Pandas, SQL, and Rust to normalize inconsistent data structures for downstream analytics and visualization tools.

- Adopted DevSecOps principles; implementing CI/CD pipelines with Azure DevOps and Git for configuration management, declarative deployments, and vulnerability scanning; reducing operational risk and minimizing technical debt.

- Managed a team of 14 developers using Agile delivery processes including sprint planning, backlog refinement, release planning, and reporting to ensure on-time, stable releases, rapid incident response, and sustained team performance.

- Conducted user requirements investigation with analysts and client leadership, translating operational needs into clear engineering tasks, technical designs, program increments, and release milestones.

- Maintained the project's knowledge base in Confluence, authoring technical documentation, API specifications, requirements definitions, on-boarding materials, and end-user guides &mdash; establishing knowledge continuity throughout the project lifetime.
